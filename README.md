# somes

The source code of the somes repository.

## The base structure is as follows:
- **`somes-api`**

    Is the backend of somes. It depends mainly on the [`axum`] web framework. Several client / frontend side applications can use the api to provide an individual interface for somes (e.g. website, desktop apps, smartphone apps).
    Later, this will also host the files generated by the frontend.

- **`sommes-common-lib`**

    A rust library containing common structures and code that should be used interoperability between the backend and the frontend applications, which are written in Rust. (e.g. `LoginInfo` is probably the same for both ends.)

- **`somes-frontend`**

    The somes frontend. It is written in [`Yew`], however, this may change during the development process.


[`axum`]: https://github.com/tokio-rs/axum
[`yew`]: https://github.com/yewstack/yew

## Building and running of the development suite

### Prerequisites
- **Rust** v1.64 or higher (with cargo)
- preferably `Visual Studio Code` with the `rust-analyzer` extension
- **trunk** -> is used to launch a debug version of the frontend

    install via `cargo install --locked trunk`

### Running (debug)
- **somes-api**: `cargo run --release`
- **somes-frontend**: `trunk serve --release`

## Contributing

Keep in mind that many different people may work on this project in the course of a few years.
Therefore, the code quality should be as high as possible.

How is this achieveable? General tips and conventions:
- document everything
- write unit tests
- keep functions small
- does a function really do the thing you described in the name?
- use simililar code style and naming -> all common structs should end with `Info`. (e.g. `LoginInfo`, `UserInfo`, `RegulationInfo`, ...)
